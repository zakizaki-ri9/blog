# ブログ再構築計画

## 概要

パフォーマンスと拡張性を重視したブログの再構築計画。
既存記事（`src/content/blog/`）を維持しつつ、ホスティング・画像配信・検索機能を刷新する。

## 技術選定（確定）

| 領域 | 技術 | 選定理由 |
|---|---|---|
| フレームワーク | Astro v5 + MDX（継続） | 既存構成を維持。パフォーマンス面で問題なし |
| ホスティング | **Cloudflare Pages** | 無制限帯域・グローバルCDN・SSR対応可・無料 |
| 画像保管庫 | **microCMS** | ストレージ無制限・超過時API停止（課金なし）・日本語UI |
| 画像方式 | **ビルド時ダウンロード** | microCMS → Astro Build → 静的サイト組込。訪問者は CDN から配信 |
| 検索機能 | **Pagefind** | 完全無料OSS・ビルド時インデックス・クライアントサイド検索 |

## 設計判断の記録

### ビルド時ダウンロード方式の採用理由

画像を microCMS に保管しつつ、ビルド時に API 経由でダウンロードし、
Astro の `<Image>` コンポーネントで最適化（WebP/AVIF変換）した上で静的サイトに組み込む。

**メリット**:
- 訪問者は Cloudflare Pages CDN から配信 → microCMS への直接アクセスなし
- 攻撃を受けても microCMS に課金が発生しない
- Astro の組み込み画像最適化が利用可能

### microCMS 選定理由

- **超過時はAPI停止**（翌月リセット）→ 従量課金が一切発生しない
- **メディアストレージ無制限**
- **日本語の管理UI・ドキュメント・サポート**
- **Astro + microCMS の連携実績**が日本語コミュニティで豊富
- imgix 基盤の画像最適化API

### Cloudflare Pages 選定理由

- **無制限の帯域幅・リクエスト**で将来的なアクセス増にも無料で対応
- **グローバル CDN** による低 TTFB
- 将来 SSR が必要になった場合も Workers で対応可能

### Pagefind 選定理由

- **完全無料**（OSS）かつ**追加インフラ不要**
- Rust製で高速、インデックスをチャンク分割してオンデマンドロード
- 静的サイトとの相性が最高
