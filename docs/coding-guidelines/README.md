# コーディングガイドライン

このディレクトリには、Astro/TypeScript プロジェクトにおけるコーディングとコードレビューのガイドラインをまとめています。

## 目的

実装時およびコードレビュー時に参照すべき、技術的な詳細ガイドラインを提供します。

> **重要**: エージェントが常に参照すべき要点は [`.cursor/rules/coding.mdc`](../../.cursor/rules/coding.mdc) に記載されています。このディレクトリは詳細な説明、実装例、チェックリストを提供します。

エージェント開発の全体的なガイドラインやTDDの原則については、[`AGENTS.md`](../../AGENTS.md) を参照してください。

## ドキュメント構成

### [アーキテクチャ設計](./architecture.md)

テスト容易性を確保するためのアーキテクチャ設計原則をまとめています。

- 依存逆転（DIP）、Functional Core/Imperative Shell
- 推奨ディレクトリ構成とポート例
- テストダブル運用、契約テスト
- Astroとの境界ポリシー、副作用の隔離
- 回帰防止テクニック

### [Astro/TypeScript](./astro-typescript.md)

Astro と TypeScript に特化したコーディングガイドラインです。

- Astro固有のチェック項目（コンポーネント、静的生成、画像最適化、SEO、パフォーマンス）
- TypeScriptチェック項目（型安全性、nullチェック、型定義、インポート/エクスポート）
- コーディングと設計の指針

### [セキュリティ](./security.md)

セキュリティに関するチェック項目とベストプラクティスです。

- XSS対策、環境変数管理
- 依存関係の脆弱性管理
- 機密情報のハードコーディング回避
- 入力値検証、認証・認可
- その他のセキュリティ考慮事項

### [コード品質](./code-quality.md)

コード品質とレビュー時の注意事項をまとめています。

- 可読性、保守性、パフォーマンス、エラーハンドリング
- レビュー時の注意事項

## 使用方法

各ドキュメントは独立して参照できますが、以下の順序で読むことを推奨します：

1. **アーキテクチャ設計** - プロジェクト全体の設計方針を理解
2. **Astro/TypeScript** - 実装時の具体的なガイドライン
3. **セキュリティ** - セキュリティチェック項目
4. **コード品質** - レビュー時の品質チェック
