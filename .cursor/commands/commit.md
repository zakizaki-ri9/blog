# commit

**目的**: 適切な粒度でのコミット作成
**補足**: 必ず日本語の説明でコミットする
1. **分析**: `git status` と `git diff` で変更内容を把握する。
2. **ループ実行**: 変更を「論理的な単位（機能・ファイルごと）」に分割し、以下を繰り返す。
   - `git add {対象ファイル}`
   - コミットメッセージを作成 (Conventional Commits準拠: `feat:`, `fix:`, `refactor:` 等 + 日本語の説明)
   - `git commit -m "{メッセージ}"`
3. **完了条件**: 全ての変更がコミットされるまで繰り返す。
