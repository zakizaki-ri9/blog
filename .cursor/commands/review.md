# review

**目的**: コード品質チェック（ESLint、TypeScript型チェック、セキュリティチェック）の実行と、厳格なコードレビューの実施
**補足**: すべてのチェックが成功した場合のみ続行可能。失敗時はエラーメッセージを表示し、フローを停止する。
**参照ドキュメント**: `docs/coding-guidelines.md` を必ず参照し、ガイドラインに基づいた厳格なレビューを実施する。

**実行フロー**:
1. **ガイドラインの確認**: `docs/coding-guidelines.md` を読み込み、レビュー基準を確認する。
2. **ESLintチェック**: `pnpm lint` を実行し、コードスタイルとベストプラクティスを検証
   - 失敗時: エラーメッセージを表示し、`pnpm lint:fix` の実行を推奨。フローを停止。
   - 成功時: ガイドラインに基づき、コードスタイル、可読性、保守性の観点から追加レビューを実施。
3. **TypeScript型チェック**: `pnpm type-check` を実行し、型安全性を検証
   - 失敗時: エラーメッセージを表示し、型エラーの修正を推奨。フローを停止。
   - 成功時: ガイドラインの「TypeScriptチェック項目」に基づき、型安全性、nullチェック、型定義の適切性を厳格にレビュー。
4. **セキュリティチェック**: `pnpm security-check` を実行し、依存関係の脆弱性を検証
   - 失敗時: エラーメッセージを表示し、脆弱性の確認と修正を推奨。フローを停止。
   - 成功時: ガイドラインの「セキュリティチェック項目」に基づき、XSS対策、環境変数管理、機密情報のハードコーディング回避、入力値検証などを厳格にレビュー。
5. **Astro固有のレビュー**: ガイドラインの「Astro固有のチェック項目」に基づき、コンポーネントの使用、静的生成/SSRの使い分け、画像最適化、SEO、パフォーマンスを厳格にレビュー。
6. **コード品質レビュー**: ガイドラインの「コード品質チェック項目」に基づき、可読性、保守性、パフォーマンス、エラーハンドリングを厳格にレビュー。
7. **レビュー結果の報告**: 発見された問題点を重要度順に整理し、具体的な修正案とともに報告する。
8. **成功時**: すべてのチェックとレビューが成功したことを確認し、次のステップに進む。
