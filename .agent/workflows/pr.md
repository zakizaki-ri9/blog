# PR Workflow

このワークフローは、現在のブランチをプッシュし、GitHub上でPRを作成または更新します。
**前提**: `gh` CLI が認証済みであること。

## 1. Push

現在のブランチをリモートにプッシュします。

1. Push実行
```bash
git push origin HEAD
```
※ upstreamが設定されていない場合は、gitの出力に従って `-u` 付きで実行してください。

## 2. 既存PRの確認

現在のブランチに関連するPRがあるか確認します。

// turbo
1. PRリスト確認
```bash
gh pr list --head "$(git branch --show-current)"
```

## 3. PRの作成・更新

### PRが存在しない場合

PRを新規作成します。

1. PR作成
```bash
gh pr create
```
- **Title**: 実装内容の要約（**日本語**）
- **Body**: 変更点の詳細リスト（**Markdown形式、日本語**）
- 手順: コマンド実行後、対話モードまたはフラグで入力してください。

### PRが既に存在する場合

必要に応じてPRの内容を更新します（タイトルや本文の修正が必要な場合）。
コードの変更だけであればPushのみで十分ですが、説明を追記したい場合は以下を使用します。

1. PR編集 (任意)
```bash
gh pr edit
```
- 内容を適宜修正・追記してください（**日本語**）。
